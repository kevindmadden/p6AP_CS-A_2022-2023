Project #01 - Automated Mad-Lib

Description:

Write a Mad-Lib. Your program should ask the user to enter words and numbers and then print out the Mad-Lib with the blanks filled in.

Requirements:

  1. Your Mad-Lib program should have at least 6 blanks.
  2. At some point, your program must ask the user to fill in an integer value for a blank, a decimal (double) value for a blank, and a word (String) for a blank.
  3. In your Mad-Lib, there should be at least one point in which a user switches from typing in a word in the previous blank to a number in the current blank.
  4. Similarly, there should be at least one point in which a user switches from typing in a number to typing in a word.
  5. You need to write your own original Mad-Lib for this assignment.
  6. Your program must ask for all of the blanks first and then print out the entire Mad-Lib once the user has typed in the values for all of the blanks.
  7. Your code should be properly formatted. Indentation should be consistent throughout your code. Variable names should make sense and be clearly named.

Reminder:

  • When you call keyboard.nextInt() or keyboard.nextDouble(), this will not move past the hidden linebreak character at the end of a line.
  • If you switch from asking for a number to be inputted to asking for a String, then you will need an extra "dummy" call to keyboard.nextLine() to consume the hidden line break character.
